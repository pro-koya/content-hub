# Lift.ly Simple Workout Log — X(Twitter) マーケティング用コンテキスト

## What is Liftly?
Liftly is a lightweight habit-tracking app that helps users build daily routines.
It focuses on simplicity — no bloat, no social features, just you and your habits.

## 1. アプリの仕組み

### 概要
Lift.ly は Flutter 製のクロスプラットフォーム（iOS / Android）フィットネス記録アプリ。ジムや自宅トレーニング中に、素早くセット・重量・レップ数を記録できることに特化している。

### 技術アーキテクチャ
- **ローカルファースト設計**: すべてのデータは端末内の SQLite に保存。ネットワーク接続なしで全機能が使える
- **クラウド同期（Pro限定）**: Supabase（PostgreSQL）によるクラウドバックアップ。ローカル → サーバーへの同期
- **状態管理**: Riverpod によるリアクティブな UI 更新
- **対応言語**: 日本語・英語の完全ローカライズ対応

### データの流れ
1. ユーザーが種目・重量・回数を入力
2. ローカル SQLite に即座に保存
3. 前回の記録を自動参照し、次のセットを素早く入力
4. セッション完了時に履歴として記録
5. （Pro）クラウドに自動同期

## 2. 機能一覧

### コア機能（無料で全て利用可能）

| 機能 | 説明 |
|------|------|
| **ワークアウト記録** | 種目選択 → 重量・回数を入力。1セット5秒以内で記録可能 |
| **前回記録のワンタップコピー** | 前回のセットをワンタップで複製。「前回を再現」で全セット一括コピーも可能 |
| **レストタイマー** | アプリ内蔵タイマー。バックグラウンド通知・音・バイブレーション対応。アプリを切り替える必要なし |
| **カレンダー＆履歴** | 月間カレンダー表示でトレーニング日を可視化。全記録テーブル表示も対応 |
| **種目管理** | 100以上の標準種目 + カスタム種目の追加。部位別フィルター・検索機能 |
| **進捗グラフ** | 種目別の重量推移・回数推移・ボリューム推移をラインチャートで表示 |
| **目標設定＆達成** | 種目ごとに重量・回数・ボリュームなどの目標を設定。一括登録・一括編集（削除・再計算・カスタム倍率）に対応。達成時にお祝いメッセージ |
| **体重記録** | 体重の推移をグラフで管理 |
| **ルーティン（テンプレート）** | よく行うトレーニングメニューをテンプレートとして保存・読み込み |
| **バックアップ＆復元** | JSON形式でデータを丸ごとエクスポート。CSV出力にも対応 |
| **テーマカスタマイズ** | 5種類のプリセット + カスタムカラーピッカーで自分好みの外観に |
| **チュートリアル** | 初回起動時のインタラクティブガイドで迷わず使い始められる |

### Pro 限定機能（サブスクリプション）

| 機能 | 説明 |
|------|------|
| **クラウド同期** | Supabase によるデータのクラウドバックアップ。端末を変えてもデータを引き継げる |
| **広告非表示** | アプリ内のバナー広告を完全に非表示にし、クリーンな体験を提供 |

### 広告の配置方針
- ワークアウト記録画面・ホーム画面には広告を表示しない（記録の集中を妨げない設計）
- 設定・履歴・体重画面など、閲覧系の画面にのみ控えめにバナーを配置

## 3. 開発背景

### なぜこのアプリを作ったのか
- 既存のワークアウトアプリは多機能すぎて、セット間の短い休憩時間に素早く記録するのが難しかった
- 「前回何キロで何回やったっけ？」をすぐ確認して、次のセットに集中したかった
- シンプルだけど必要な機能は揃っている、そんなアプリが欲しかった

### 開発の経緯
- 2026年2月に App Store に初回リリース（v1.0.0）
- 個人開発プロジェクトとして、実際にジムで使いながら改善を重ねている
- ユーザーフィードバックとApp Storeレビューを元に継続的に機能追加・改善中
- 現在 v1.0.6（データベーススキーマ v19、11以上のエンティティ、20のフィーチャーモジュール）

### 開発のこだわり
- **オフラインファースト**: ネットがなくても一切困らない設計
- **記録画面の UX 最優先**: 広告も通知も記録中は邪魔にならない設計
- **ローカライズ**: 日本語・英語ネイティブ対応（.arb ファイルベースで拡張可能）

## 4. 特徴（差別化ポイント）

### 「次のセットを数秒で記録」がコアバリュー
> ホーム画面から2タップで記録開始。1セットの入力は5秒以内。セット間の集中を途切れさせない。

### 5つの主な強み

1. **スピード最優先の入力設計**
   - 片手操作を前提とした親指ゾーン最適化
   - タップ数・スワイプ数を最小限に
   - 前回記録のワンタップコピーで入力の手間を大幅削減

2. **前回記録が常に見える**
   - 記録画面上に前回のセット内容を直接表示
   - 画面遷移なしで「前回の自分」と比較しながらトレーニング
   - 「前回を再現」ボタンで全セット一括コピー

3. **アプリ内蔵レストタイマー**
   - 別のタイマーアプリを開く必要なし
   - バックグラウンド通知でアプリを閉じても安心
   - 音 + バイブレーションで確実にお知らせ
   - カスタマイズ可能なプリセット（60秒/90秒/120秒）

4. **オフライン完結で安心**
   - アカウント登録不要で即使い始められる
   - 全データはデバイスに保存、プライバシーも安心
   - クラウド同期は必要な人だけ（Pro）

5. **無料でもほぼ全機能使える**
   - 記録・履歴・グラフ・タイマー・目標・ルーティン・バックアップ・テーマ変更、すべて無料
   - Pro は「クラウド同期」と「広告非表示」のみ
   - 無料ユーザーにも快適に使ってもらうフリーミアム設計

## 8. 最新のアップデート情報（2026年3月）

### 目標まわりの大幅強化

#### 目標の一括登録
- **対象**: トレーニング履歴があり、かつまだ目標を設定していない種目だけを一覧表示
- **流れ**: 目標一覧で「一括登録」→ 係数コース（1.1×／1.2×／1.3×／1.5×）を選択 → 種目をチェックで複数選択 → 実行
- **計算**: 各種目の「現在の最高記録 × 選択した倍率」で目標値を自動計算。重量・ボリュームは 2.5 kg 単位でキリよく繰り上げ
- **補足表示**: 「目標値は現在の最高記録から選択した倍率で計算」「表示されている種目は過去にトレーニング履歴があるもののみ」を画面内で説明

#### 目標の一括編集
- **削除**: 既存の目標をチェックで複数選択し、「選択した目標を削除」で一括削除
- **再計算**: 選択した目標について、固定コース（1.1×〜1.5×）または**カスタム倍率**（0.1〜10の範囲で自由入力）で「過去最高 × 倍率」を再計算して上書き
- **過去最高の表示**: 一括編集画面の各種目に「過去最高: ○○ kg」など、種目タイプ・単位に合わせた補足を表示。カスタム倍率を決める際の参考になる

#### その他の改善
- **結果メッセージ**: 一括登録・一括編集でスキップがあった場合、「○件設定（△件スキップ）」とスキップ理由（記録タイプに合うベスト値が無い種目）を表示
- **目標一覧のヘッダー**: ボタン文言を記録画面と同様の前景色で表示し、並び順を「目標追加 → 一括登録 → 一括編集」に統一

### マーケティングでの使い方
- 「過去の記録から一括で目標を自動計算」「1.1倍〜1.5倍で無理のない目標からマッチョ目標まで選べる」
- 「目標の一括編集で、削除や倍率の再計算がまとめてできる」「カスタム倍率で自由な目標設定が可能」
- 「各種目に過去最高を表示するので、次の目標を立てやすい」
