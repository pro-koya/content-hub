# X Post Growth Plan

## 目的
- フォロワー数を増やし、AI駆動開発・農業・ハイブリッドキャリア領域で継続的に見つけられるアカウントにする
- 単発の宣伝ではなく、「最新情報を実務に翻訳してくれるアカウント」として認知される
- 投稿改善の仮説検証を回し、徐々に精度を上げる

## 現状の課題
1. `liftly` 固定で、筋トレ・個人開発文脈に強く寄っている
2. ダイジェストの最新情報が投稿生成に使われていない
3. 投稿ごとの仮説やメタデータが残らず、改善理由を追えない
4. 分析軸が少なく、何が伸びたのかを学習しづらい
5. 1日1投稿で検証速度が遅い

## 目指す状態
- プロジェクトごとに投稿方針を切り替えられる
- 最新ダイジェストからトピック候補を拾って投稿に反映できる
- 投稿ごとに `content_pillar` `source_category` `target_audience` `hypothesis` などを保存する
- 実験分析で、どの切り口が反応を得たか見える

## 優先順位

### P1
- `project` 切り替え対応
- `content_hub` 向けのコンテキスト新設
- ダイジェスト連携によるトピック選定
- 投稿メタデータ保存

### P2
- 分析軸の拡張
- lint のプロジェクト対応
- dry-run でも prompt を確認できるようにする

### P3
- 投稿本数や時間帯の実験設計
- フォロー率やプロフィール遷移など、より強い獲得指標の導入
- 候補投稿の複数保存と A/B 運用

## このパッチで実施したこと
- `content_hub` をデフォルトプロジェクトとして追加
- X投稿生成を `project` 指定対応に変更
- `docs/digests/latest.md` から重点トピックを抽出し、prompt に注入
- 投稿ごとにメタデータを `output` / `posted_tweets.json` / `post_metrics.json` に保存
- 分析軸に `content_pillar` `source_category` `target_audience` を追加

## 次の改善候補
1. `xpost:generate` で複数候補を保存し、手動レビューできるようにする
2. ダイジェストトピックのスコアリングに「日本市場」「保存価値」「議論性」を入れる
3. 週次でプロンプトそのものを自動改善する仕組みを入れる
